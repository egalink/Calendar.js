calendar.js

A simple, minimal, powerful and fully customizable event calendar plugin without library dependencies.

Features:

  • Weighs only ~6KB minified and Gzip’ed.

  • Doesn’t require any external libraries.

  • You can easily add or remove events from calendar.

  • Completely customizable.

  • Works in all modern desktop and mobile browsers, including IE 9 and up.



Usage Instructions

Following the steps below you will be able to get the plugin up and running. If you notice any bugs, please post them to GitHub issues.


Add markup

<!-- put this line in anywhere of your <body> in your HTML -->
<div id="calendar"></div>

Link & Hook up the plugin

All ready-to-use files are located in the dist/ directory.

<!-- put this right before the </body> closing tag -->
<link href="dist/calendar.min.css" rel="stylesheet"/>
<script src="dist/calendar.min.js"></script>
<script>
    var calendar = new Calendar(document.getElementById('calendar'));
</script>

A calendar will appear after do this correctly.


Public methods

Once you have a reference to Calendar instance following methods are available.

// change calendar to next month:
calendar.nextMonth();
// change calendar to previous month:
calendar.prevMonth();
// change calendar to current month:
calendar.currMonth();
// add events to calendar.
// each user event has dinamically asigned to a unique ID to identify the registry:
calendar.addEvents({
    id : 1,
    datetime : "2018-06-15 18:45:00",
    title : "Event title.",
    description : "A brief description for the event.",
    status : 65
}, ...);
// Remove a user-defined event from the calendar by
// passing the event ID as a parameter.
calendar.removeEvent(id);
// get all user events or get events filtered by a valid "YYYY-MM-DD" date (optional):
var events = calendar.getEvents(date);
// parse a valid 'Y-m-d' or 'Y-m-d H:i:s' date string to js Date Format
// and returns new Date object or FALSE if date string is invalid.
var dateOb = calendar.dateParse(date);
// assign new custom function to "onDayClick" event.
calendar.onDayClick(function (el, date, events) {
    console.log(arguments);
});

Customization

Overwrite the default settings as shown below.

<script>
    var settings = {
        // "string": The name of the CSS calendar theme.
        theme: 'calendar',
        // {object}: Language options.
        i18n: {
            // "string": Text for previous month button.
            prevMonth: '<',
            // "string": Text for next month button.
            nextMonth: '>',
            // [array]: Name for each month shown on the calendar.
            months: ['January', 'February', 'March', 'April', 'May', 'June', 'July', 'August', 'September', 'October', 'November', 'December'],
            // [array]: Name for each day of the week shown on the calendar.
            weekdays: ['Sun', 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat']
        },
        // [array]: All user defined events.
        // To generate an event in the calendar, you can pass a object-key-value
        // with all values ​​as you need.
        events: [
            {
                // "string" | integer: id is optional.
                // if you don't provide a id in each record, the plugin will dynamically
                // assign a unique id in each one.
                id: 1,
                // "string": a valid datetime: "YYYY-MM-DD HH:MM:SS" string (required).
                datetime: "2018-06-15 18:45:00"
                // all other data ...
                title: "Event title.",
                description: "A brief description for the event."
                status: 64,
                // ...
            }
        ],
        // integer: Start the week off on Sunday (0), Monday (1), etc.
        // Sunday is the default.
        weekOffset: 0,
        // function: It is fired when a calendar day is clicked.
        onDayClick: function (el, date, events) {}
    };

    // pass the object with your configuration as a second parameter:
    var calendar = new Calendar(document.getElementById('calendar'), settings);
</script>
